## 前置准备

### 1. 购买云服务器

推荐配置：

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 1 核 | 2 核 |
| 内存 | 1 GB | 2 GB |
| 硬盘 | 20 GB | 40 GB |
| 带宽 | 1 Mbps | 3 Mbps |
| 系统 | Ubuntu 20.04+ | Ubuntu 22.04 LTS |

**推荐云服务商：**
- 阿里云：https://www.aliyun.com
- 腾讯云：https://cloud.tencent.com
- 华为云：https://www.huaweicloud.com

**购买步骤：**
1. 注册账号并完成实名认证
2. 购买 ECS/CVM 云服务器
3. 选择 Ubuntu 22.04 LTS 系统镜像
4. 设置 root 密码（务必记住！）
5. 获取服务器公网 IP 地址

### 2. 连接服务器

**Windows 用户：**

1. 下载并安装 [MobaXterm](https://mobaxterm.mobatek.net/download.html) 或 [XShell](https://www.netsarang.com/zh/xshell/)
2. 打开软件，新建 SSH 连接
3. 填入服务器 IP、端口 22、用户名 root、密码
4. 点击连接

**Mac/Linux 用户：**

打开终端，输入：
```bash
ssh root@你的服务器IP
```

输入密码（不会显示，直接输入后回车）。

### 3. 准备域名（可选）

如果有域名：
1. 在域名服务商处添加 A 记录，指向服务器 IP
2. 等待 DNS 解析生效（约 10 分钟）

没有域名也可以用 IP 地址直接访问。

## 一键部署（推荐）

这是最简单的部署方式，脚本会自动完成所有配置。

### 步骤 1：上传项目到服务器

**方式 A：使用 Git（推荐）**

在服务器上执行：
```bash
# 安装 Git
apt update && apt install -y git

# 克隆项目
git clone https://github.com/DeXuan/soulquad-web.git /var/www/soulquad
```

**方式 B：使用 SFTP 上传**

1. 用 MobaXterm 连接服务器
2. 左侧面板找到本地项目文件夹
3. 右键上传到 `/var/www/soulquad`

**方式 C：使用 scp 命令**

在本地电脑执行：
```bash
scp -r ./soulquad-web root@你的服务器IP:/var/www/soulquad
```

### 步骤 2：执行部署脚本

```bash
# 进入项目目录
cd /var/www/soulquad

# 给脚本执行权限
chmod +x deploy/deploy-to-server.sh

# 执行一键部署
sudo ./deploy/deploy-to-server.sh
```

脚本会自动完成以下操作：
1. ✅ 安装 PostgreSQL 数据库
2. ✅ 安装 Node.js 运行环境
3. ✅ 安装 PM2 进程管理器
4. ✅ 创建数据库和用户
5. ✅ 安装项目依赖
6. ✅ 构建前端代码
7. ✅ 配置环境变量
8. ✅ 启动服务

### 步骤 3：访问应用

部署完成后，你会看到类似输出：

```
========================================
  部署完成!
========================================

访问地址: http://你的服务器IP
API地址: http://你的服务器IP:3000

数据库信息:
  Host: localhost
  Port: 5432
  Database: soulquad
  User: soulquad_user
  Password: SoulQuad2024!

登录账号:
  alice, bob, carol, david, emma, frank
  或 user_f_1, user_f_2, ... user_m_1, ...
  密码: demo123
```

在浏览器打开 `http://你的服务器IP` 即可访问！

## 手动部署

如果你想了解更多细节，或者一键部署失败，可以手动部署。

### 步骤 1：系统更新

```bash
apt update && apt upgrade -y
```

### 步骤 2：安装 Node.js

```bash
# 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 验证安装
node -v   # 应显示 v20.x.x
npm -v    # 应显示 10.x.x

# 安装 PM2（进程管理器）
npm install -g pm2
```

### 步骤 3：安装 PostgreSQL

```bash
# 安装 PostgreSQL
apt install -y postgresql postgresql-contrib

# 启动服务
systemctl start postgresql
systemctl enable postgresql
```

### 步骤 4：配置 PostgreSQL

```bash
# 切换到 postgres 用户
su - postgres

# 创建数据库和用户
psql -c "CREATE DATABASE soulquad;"
psql -c "CREATE USER soulquad_user WITH ENCRYPTED PASSWORD 'SoulQuad2024!';"
psql -c "GRANT ALL PRIVILEGES ON DATABASE soulquad TO soulquad_user;"
psql -c "ALTER DATABASE soulquad OWNER TO soulquad_user;"

# 退出 postgres 用户
exit
```

**配置远程连接（可选）：**

如果需要从本地电脑连接数据库：

```bash
# 编辑 pg_hba.conf
nano /etc/postgresql/16/main/pg_hba.conf

# 在文件末尾添加：
host    all    all    0.0.0.0/0    md5

# 编辑 postgresql.conf
nano /etc/postgresql/16/main/postgresql.conf

# 找到 #listen_addresses = 'localhost'
# 改为：listen_addresses = '*'

# 重启 PostgreSQL
systemctl restart postgresql
```

### 步骤 5：上传项目

```bash
# 创建目录
mkdir -p /var/www/soulquad

# 上传项目文件（或 git clone）
# ...
```

### 步骤 6：安装项目依赖

```bash
cd /var/www/soulquad

# 清理旧文件
rm -rf node_modules package-lock.json

# 安装依赖
npm install --omit=dev

# 安装构建工具
npm install vite typescript --save-dev
```

### 步骤 7：配置环境变量

```bash
# 创建 .env 文件
cat > .env << 'EOF'
NODE_ENV=production
PORT=3000

PG_HOST=localhost
PG_PORT=5432
PG_DATABASE=soulquad
PG_USER=soulquad_user
PG_PASSWORD=SoulQuad2024!

ALLOWED_ORIGINS=http://你的服务器IP
TOKEN_SECRET=SoulQuad-Secret-Key-2024
EOF
```

### 步骤 8：构建前端

```bash
# 修改 API 地址为相对路径
sed -i "s|const API_BASE = 'http://localhost:3001/api';|const API_BASE = '/api';|" src/services/api.ts

# 构建
node ./node_modules/vite/bin/vite.js build
```

### 步骤 9：启动服务

```bash
# 使用 PM2 启动
pm2 start server/index.js --name soulquad

# 保存进程列表
pm2 save

# 设置开机自启
pm2 startup
```

### 步骤 10：安装 Nginx

```bash
# 安装 Nginx
apt install -y nginx

# 创建配置文件
cat > /etc/nginx/sites-available/soulquad << 'EOF'
server {
    listen 80;
    server_name _;  # 替换为你的域名

    # 前端静态文件
    root /var/www/soulquad/dist;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # 前端路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # Socket.IO 代理
    location /socket.io/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
EOF

# 启用配置
ln -sf /etc/nginx/sites-available/soulquad /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重启 Nginx
systemctl restart nginx
```

### 步骤 11：开放防火墙

```bash
# 开放 HTTP 和 HTTPS 端口
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 3000/tcp

# 重新加载防火墙规则
ufw reload
```

### 步骤 12：访问应用

在浏览器打开 `http://你的服务器IP`

## 部署后配置

### 配置 SSL 证书（HTTPS）

**使用 Let's Encrypt 免费证书：**

```bash
# 安装 Certbot
apt install -y certbot python3-certbot-nginx

# 申请证书（替换为你的域名）
certbot --nginx -d your-domain.com -d www.your-domain.com

# 自动续期测试
certbot renew --dry-run
```

**使用自签名证书（测试用）：**

```bash
chmod +x deploy/generate-ssl.sh
./deploy/generate-ssl.sh
```

### 修改默认密码

**修改数据库密码：**

```bash
su - postgres
psql -c "ALTER USER soulquad_user WITH PASSWORD '你的新密码';"
exit
```

然后修改 `.env` 文件中的 `PG_PASSWORD`。

**修改 JWT 密钥：**

编辑 `.env`，修改 `TOKEN_SECRET` 为一个随机字符串。

### 配置域名

如果有域名，修改 Nginx 配置：

```bash
nano /etc/nginx/sites-available/soulquad

# 修改 server_name
server_name your-domain.com www.your-domain.com;

# 重启 Nginx
systemctl restart nginx
```

## 常见问题排查

### 问题 1：无法访问网站

**检查服务状态：**
```bash
# 检查 PM2 进程
pm2 status

# 检查 Nginx 状态
systemctl status nginx

# 检查端口是否监听
ss -tlnp | grep -E '80|3000'
```

**检查防火墙：**
```bash
# 查看防火墙状态
ufw status

# 如果使用云服务器，还需要在控制台配置安全组
# 开放端口：80, 443, 3000, 22
```

### 问题 2：数据库连接失败

```bash
# 检查 PostgreSQL 是否运行
systemctl status postgresql

# 测试数据库连接
psql -h localhost -U soulquad_user -d soulquad

# 查看 PostgreSQL 日志
tail -f /var/log/postgresql/postgresql-*-main.log
```

### 问题 3：前端白屏

```bash
# 检查 dist 目录是否存在
ls -la /var/www/soulquad/dist

# 重新构建
cd /var/www/soulquad
node ./node_modules/vite/bin/vite.js build

# 检查 Nginx 配置
nginx -t
```

### 问题 4：Socket.IO 连接失败

```bash
# 检查 WebSocket 代理配置
cat /etc/nginx/sites-available/soulquad | grep -A5 "socket.io"

# 查看后端日志
pm2 logs soulquad
```

### 问题 5：内存不足

```bash
# 查看内存使用
free -h

# 如果内存不足，添加 swap
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

```bash

pm2 status

pm2 logs soulquad

# 重启应用
pm2 restart soulquad

# 停止应用
```

### 数据库备份

```bash
# 创建备份目录
mkdir -p /backup

# 备份数据库
pg_dump -U soulquad_user -d soulquad > /backup/soulquad_$(date +%Y%m%d).sql

# 自动备份（每天凌晨 3 点）
crontab -e
# 添加：0 3 * * * pg_dump -U soulquad_user -d soulquad > /backup/soulquad_$(date +\%Y\%m\%d).sql
```

### 数据库恢复

```bash
# 恢复数据库
psql -U soulquad_user -d soulquad < /backup/soulquad_20260517.sql
```

## 安全建议

### 1. 修改 SSH 端口

```bash
nano /etc/ssh/sshd_config

# 修改 Port 22 为其他端口，如 Port 2222
# 重启 SSH
systemctl restart sshd
```

### 2. 禁用 root 登录

```bash
# 创建普通用户
adduser deploy
usermod -aG sudo deploy

# 修改 SSH 配置
nano /etc/ssh/sshd_config
# PermitRootLogin no

# 重启 SSH
systemctl restart sshd
```

### 3. 配置 fail2ban

```bash
apt install -y fail2ban
systemctl enable fail2ban
systemctl start fail2ban
```

### 4. 定期更新系统

```bash
apt update && apt upgrade -y
```

## 性能优化

### 1. 启用 Gzip

已在 Nginx 配置中启用。

### 2. 配置缓存

### 3. 数据库优化

```bash
# 编辑 PostgreSQL 配置
```
